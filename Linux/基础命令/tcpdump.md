# tcpdump

`tcpdump` 是一款用于在网络上转储（抓取和分析）网络流量的专业命令行工具 。它能够监听指定的网络接口，并打印出与布尔表达式相匹配的数据包的内容描述 。



`tcpdump` 的命令行参数在结构上可以清晰地划分为**两个核心部分**：**选项（Options）**和**表达式（Expression）** 。

**选项（Options）**：用于控制 `tcpdump` 自身的运行机制、抓包方式以及输出格式。

- 选项可以控制数据的输入输出流向，例如使用 `-w` 将原始数据包写入文件，或使用 `-r` 从文件中读取 。
- 选项可以指定程序的系统级行为，例如使用 `-i` 监听特定的网络接口 。
- 选项还能调节信息显示的细节，例如使用 `-n` 阻止将主机地址转换为名称，从而避免 DNS 查找 。



**表达式（Expression）**：充当核心的过滤器，用于精确选择哪些数据包将被捕获或处理 。

- 如果命令中未提供任何表达式，网络上的所有数据包都将被转储 。
- 如果指定了表达式，`tcpdump` 会对抓取到的每一个数据包进行校验，只有当数据包与表达式匹配（即表达式逻辑为“真”）时，该数据包才会被处理并输出 。





## 选项

### 指定接口

在未明确指定监听网卡的情况下，tcpdump 并不会盲目地抓取所有网卡的流量。它的默认行为是在系统的网络接口列表中进行搜索，寻找**编号最小**且处于**已配置并启动状态**的接口，同时它会**主动排除环回（loopback）接口** 。

- **-D**：打印系统中所有可用的网络接口列表，每个接口会显示编号和名称，可用于 `-i` 参数引用。
- **-L**：列出指定接口在指定模式下（是否监控模式）支持的数据链路类型（如：Ethernet、PPPOE 等），然后退出。该列表可能因模式而异。

- **-i interface**：选项来强制 `tcpdump` 监听你指定的网络接口，可以使用 any 参数监听所有的接口。如：`tcpdump -i eth0`，`-i any` 。



### 指定网卡模式

tcpdump 本身并不直接与网卡硬件交互，而是通过操作系统内核接收网卡上传的数据包。因此，它能否捕获到所有经过网卡的数据包，根本上取决于网卡当前是否处于混杂模式。

**tcpdump 默认会将网卡设置为混杂模式**（例如不指定 `-p` 参数时），但混杂模式是系统级的共享状态——不仅 tcpdump，系统中其他程序（如 Wireshark、其他监控工具）同样可以修改它。这意味着 tcpdump 无法独占这一设置，也无法强制网卡退出混杂模式。即便你通过 `-p` 参数要求 tcpdump 以非混杂模式运行，如果网卡此前已被其他进程设为混杂模式，tcpdump 也无法将其恢复。因此，仅靠 tcpdump 这个工具，无法保证网卡 100% 处于非混杂模式。

在正常的非混杂模式下，网卡通常只会向操作系统传递目的 MAC 地址为本机的报文以及广播/组播报文。而进入混杂模式后，网卡会捕获流经该物理链路的所有报文，无论其目标 MAC 地址是否为本机。

- **-p**：不将网络接口置于混杂模式 。**注意，接口可能因其他原因已处于混杂模式，此选项仅阻止 tcpdump 主动启用混杂模式。**



文档中特别指出，不能将 `-p` 视作以下 BPF（伯克利数据包过滤器）表达式的缩写 ：

```
ether host {local-hw-addr} or ether broadcast
```

- 以上表达式为 tcpdump 中的表达式参数。

- **ether host {local-hw-addr}**：指定只抓取源 MAC 地址或目的 MAC 地址为本机网卡 MAC 地址的数据包。你需要把 `{local-hw-addr}` 替换为你实际网卡的 MAC 地址（例如 `00:11:22:33:44:55`）。
- **ether broadcast**：指定抓取以太网广播包 （例如 ARP 请求）。**因为即使在非混杂模式下，正常的网卡也会接收广播包，加上这一条件能最真实地还原普通模式的网络行为。**



在手册中，`tcpdump` 的 `expression`（表达式）决定了哪些数据包会被转储 ，而底层用于高效执行这些过滤规则的内核机制就是 BPF 。

你可以给它写一套规则（即表达式），当网卡收到数据包时，BPF 会在最底层极速匹配规则。只有符合规则的包才会被放行交给 `tcpdump` 程序，不符合的直接丢弃，效率极高。



### 输出格式控制

在生产环境中抓包时，为了防止 `tcpdump` 反向解析域名带来的性能瓶颈或网络请求阻塞，这两个选项几乎是必加的：

- **-n**：不将主机地址转换为名称 。这可以有效避免不必要的 DNS 查找，大幅提升抓包结果的输出速度 。
- **-nn**：在 `-n` 的基础上更进一步，不将协议和端口号等转换为名称 。
- **-N**：不打印主机名的域名修饰部分 。例如，如果使用了该标志，`tcpdump` 将只会打印 "nic" 而不是 "nic.ddn.mil" 。

- **-q**：快速输出，打印较少的协议信息，使每行输出更短。

- **-e**：在每一行输出中显示链路层头部信息，如源 MAC 地址、目的 MAC 地址、协议类型等。

- **-A**：以 ASCII 格式打印每个数据包的内容（去掉链路层头部），便于直接查看网页等文本数据。

- **-x**：以十六进制打印每个数据包的内容（不含链路层头）。若需包含链路层头，使用 `-xx`。

- **-X**：以十六进制和 ASCII 混合格式打印数据包内容（不含链路层头），`-XX` 则包含链路层头。这对分析未知协议非常有用。

- **-v, -vv, -vvv**：增加输出的详细程度。

  - `-v`：打印 IP 包的 TTL、ID、总长度、选项等信息，并启用额外的包完整性检查。

  - `-vv`：更详细，例如完整解码 NFS 和 SMB 包。

  - `-vvv`：极度详细，例如打印 Telnet 的 SB … SE 选项。

**-S**：打印 TCP 序列号时使用绝对序列号，而不是相对序列号。默认情况下，tcpdump 显示相对于初始序列号的偏移量。

**-t, -tt, -ttt, -tttt, -ttttt**：控制时间戳的打印格式

- `-t`：不打印时间戳。
- `-tt`：打印自 1970-01-01 以来的秒数。
- `-ttt`：打印当前行与上一行的时间差（微秒精度）。
- `-tttt`：打印可读的日期和时间。
- `-ttttt`：打印当前行与第一行的时间差。



### 过滤与表达式相关

**-F file**：从指定的文件中读取过滤表达式。命令行中额外给出的表达式将被忽略。

**-f**：以数字形式打印“外部”IPv4 地址（非本地地址），用于绕过 Sun NIS 服务器的缺陷。此选项依赖接口地址和掩码，在 Linux `any` 接口上可能失效。



### 文件保存与读取

这类参数用于将捕获的数据包保存到文件、从文件读取，以及管理文件轮转和压缩。

**-w file**：将原始数据包写入文件，而不是解析并打印到标准输出。该文件可用 `-r` 重新读取。

**-r file**：从文件读取数据包进行分析，文件必须是 `-w` 生成的格式。

**-C file_size**：与 `-w` 配合使用，当保存文件大小超过 `file_size`（百万字节）时，关闭当前文件并新建一个文件，新文件名为原文件名后加数字。注意：`file_size` 以 1,000,000 字节为单位，而非 1,048,576 字节。

**-G rotate_seconds**：与 `-w` 配合，每隔 `rotate_seconds` 秒轮转保存文件。文件名中可使用 `strftime` 格式，若未指定格式，则新文件会覆盖旧文件。

**-W filecount**：与 `-C` 或 `-G` 配合使用，限制创建的文件数量。达到上限后，从第一个文件开始循环覆盖（类似环形缓冲区）。同时，文件名会自动补零以正确排序。

**-z postrotate-command**：与 `-C` 或 `-G` 配合，在每个文件轮转后执行指定的命令（如 `gzip`）来压缩文件。命令会以最低优先级并行运行。



### 性能与缓冲控制

这类参数影响捕获过程中的缓冲区大小、截断长度、校验和验证等性能相关设置。

- **-c count**：捕获指定数量的包后退出，无论是否收到中断信号。
- **-s snaplen**：设置每个包的捕获长度（snaplen），默认为 65535 字节。若设置较小值（如 128），可减少处理开销，但可能截断包内容，输出中会以 `[|proto]` 提示。建议设置刚好能捕获所需协议头部的大小。
- **-B buffer_size**：设置操作系统内核捕获缓冲区的大小（单位：KB）。增大缓冲区可减少丢包，尤其是在高流量环境中。
- **-K**：不验证 IP、TCP 或 UDP 校验和。对于硬件卸载校验和的情况，避免错误地将出站 TCP 包标记为坏校验和。
- **-O**：禁用包匹配代码的优化器。仅在怀疑优化器存在 bug 时使用。
- **-U**：使输出（通过 `-w` 保存）变为“包缓冲”模式，即每收到一个包立即写入文件，而不是等待缓冲区填满。
- **-l**：使标准输出行缓冲，便于通过管道或 `tee` 实时查看输出（例如 `tcpdump -l | tee file`）。



**包匹配代码优化器**是一个对用户指定的过滤表达式（如 `host 192.168.1.1 and tcp port 80`）进行**自动优化**的组件。它位于将表达式编译为底层 BPF（Berkeley Packet Filter）指令的过程中，目的是生成更高效、执行更快的过滤代码。
